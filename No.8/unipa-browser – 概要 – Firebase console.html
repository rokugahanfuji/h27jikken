Firebase の機能
Analytics
詳細な分析情報を入手し、アプリがどれだけ使われているか測定、分析します
詳細
Authentication
サーバー側のコードを使わずに、さまざまなプロバイダのユーザーを認証し管理します
詳細
Database
接続されているすべてのクライアントでデータの保存と同期をリアルタイムで実行します
詳細
Storage
サーバー側のコードを使わずに、画像、音声、動画など、ユーザーが作成したコンテンツを保存および取得します
詳細
Hosting
セキュアなグローバル コンテンツ配信ネットワークを使用してウェブアプリやモバイル ウェブアプリを数秒でデプロイします
詳細
Cloud Functions
Firebase 機能の拡張と接続を行う、モバイルアプリのマネージド バックエンド コードを実行します
詳細
Test Lab
幅広い端末に対して Android アプリをテストします
詳細
Crashlytics
アプリの品質を損なう安定性の問題に優先順位を付けて解決する方法を理解します
詳細
Notifications
通知キャンペーンを管理し、適切なタイミングでメッセージを送信して適切なユーザーを惹き付けます
詳細
Remote Config
サーバー側の設定パラメータを使用し、アプリの動作をカスタマイズおよびテストします
詳細
Dynamic Links
アプリのインストールの有無にかかわらず、アプリ内の最適な場所にユーザーを送ります
詳細
AdMob
数多くの Google 広告主の広告を表示し、アプリからの収益をスマートに向上させます
詳細
Cloud Messaging
プラットフォーム間でメッセージを確実に送受信します
詳細
AdWords
インストールを促進し、コンバージョンに対する詳細な分析情報を得て、ターゲットを絞り込んだキャンペーンを実施することで、ユーザー層の関心を引き付けます。
詳細
App Indexing
アプリに対するオーガニック検索トラフィックを促進します
詳細
Predictions
予測される行動に基づいて動的なユーザー グループを定義します。その後、通知や Remote Config などを使ってこれらのグループをターゲットとし、コンバージョンの増加とアンインストールの削減を目指すことができます。

Gradle の使用手順
代替: Unity C++

Gradle 用の Google サービス プラグインは、ダウンロードした google-services.json ファイルを読み込みます。このプラグインを使用するよう、build.gradle ファイルを修正します。

  1. プロジェクト レベル build.gradle（<project>/build.gradle）: 

    buildscript {
  dependencies {
    // Add this line
    classpath 'com.google.gms:google-services:3.1.0'
  }
}

  2. アプリレベル build.gradle（<project>/<app-module>/build.gradle）: 

    ...
// Add to the bottom of the file
apply plugin: 'com.google.gms.google-services'

    デフォルトでアナリティクスを含める help_outline

  3. 最後に、IDE に表示されるバーの [Sync now] を押します:

パッケージ名は、通常はアプリレベルの build.gradle ファイルの applicationId です。
